## How campaigns use workflows

Campaigns are powered by [Codemod workflows](/workflows/introduction). A workflow defines the stages of a migration.

| Concept | Meaning |
| --- | --- |
| **Node** | A stage in the workflow, such as analyzing a repository or applying a transform. |
| **Step** | One action inside a node, such as running a JSSG codemod or a shell command. |
| **Task** | A runtime instance of a node. Matrix workflows can create multiple tasks. |
| **Pull request** | The branch and PR created when a task produces code changes. |

Most users do not need to edit workflow files to run a campaign. Choose a registry codemod, select repositories, and review the generated plan.

## CI checks and recovery

Generated pull requests show supported CI checks inline. Passing checks appear on the task card so you can see whether the generated PR is ready to review.

If a check fails and CI recovery is enabled, Codemod can try to repair the generated PR or report what failed. Inspect logs from the campaign page before deciding what to do next.
