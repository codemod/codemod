## What to check

Before you create the campaign, confirm:

- the selected codemod package and version
- the target repositories
- the campaign name
- any codemod arguments
- PR and branch settings
- CI recovery settings, if enabled

## Roll out safely

Start small, then expand:

1. Run the codemod against one repository.
2. Review the generated pull request.
3. Check CI status on the campaign detail page.
4. Run the campaign against more repositories.

<Tip>
  For large migrations, use sharding to split work into smaller pull requests. Group shards by directory or code owner so each team gets a reviewable slice.
</Tip>
