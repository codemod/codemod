## Common groups

| Group by | What it shows |
| --- | --- |
| **Repository** | Which repos have the signal. |
| **Directory** | Where the work is located inside a repo. |
| **Code owner** | Which team likely owns the work. |
| **Metric cardinality key** | A custom dimension emitted by JSSG, such as component name, package name, or API name. |

## Group by metric cardinality

JSSG metrics can emit cardinality keys. For example, a codemod can emit one `prop-usage` metric for each deprecated prop and attach `name = Button` or `name = Link`.

Use metric cardinality when you want a table such as:

- deprecated prop usage by component name
- API usage by imported symbol
- package usage by package name
- migration findings by rule id

## Choose the right widget type

Use a table when the grouped result has many rows. Use a timeseries chart when you want to show the same group changing over time. Use a single number when you only need the total after filters and formulas.
