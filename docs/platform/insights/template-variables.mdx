Template variables let you reuse one dashboard across different inputs. Add a variable when several widgets should share the same repository list, rule, string, or JSSG source.

## When to use variables

Use variables when:

- multiple widgets should run against the same repository list
- one dashboard should work for different teams
- the same JSSG source should be reused by several widgets
- a dashboard should compare the same metric across different values

## Variable examples

| Variable | Use it for |
| --- | --- |
| Repository | Reuse one dashboard for different repos or repo groups. |
| String | Swap a package name, component name, import path, or API name. |
| ast-grep rule | Reuse the same dashboard shape with different structural rules. |
| JSSG source | Let advanced users adjust the query logic without rebuilding the dashboard. |

## Use variables in widgets

After you create a variable, reference it from widget query controls. Repository variables are most useful when every widget on the dashboard should follow the same selected repositories.
