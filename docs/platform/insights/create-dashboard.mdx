title: 'Create an Insights dashboard'

Create a dashboard when you want one place to track a migration, dependency rollout, or code health signal.

## Create the dashboard

Go to **Insights**, create a dashboard, then name it after the outcome you want to track. Good dashboard names are specific, such as "React 19 adoption" or "Repository health".

## Add the first widget

Click **Add Widget**, then choose how you want the result to appear.

<Frame>
  <img
    src="/images/insights/widget-types-light.png"
    alt="Add widget menu with timeseries chart, single number, table view, and notes"
    className="block dark:hidden"
  />
  <img
    src="/images/insights/widget-types-dark.png"
    alt="Add widget menu with timeseries chart, single number, table view, and notes"
    className="hidden dark:block"
  />
</Frame>

| Widget | Use it for |
| --- | --- |
| **Timeseries chart** | Progress over time, such as deprecated API usage trending down. |
| **Single number** | One current value, such as files left to migrate. |
| **Table view** | Breakdowns by repository, directory, package, owner, or metric key. |
| **Notes** | Context, next actions, caveats, and links. |

## Select repositories

Every widget needs a repository source. Select one repository for a focused widget, or select multiple repositories when the result should summarize a whole migration surface.

<Frame>
  <img

  <img

<Tip>
  If several widgets should use the same repository list, add a repository template variable and reference that variable from each widget.
</Tip>

## Preview and save

After you choose a data source and configure a query, wait for the preview to reload. Save the widget when the preview shows the shape you expect.
